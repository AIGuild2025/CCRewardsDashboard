ğŸ¯ INTERVIEW TALKING POINTS

"Why is this different from other scrapers?"

1. RESILIENCE
   - Doesn't break when banks update their sites
   - Falls back gracefully: rule â†’ heuristic â†’ LLM
   - Each layer has fallback confidence thresholds

2. AUDITABILITY  
   - Every extraction timestamped & versioned
   - Confidence scores & method tracked
   - Field-level diffs show what changed

3. INTELLIGENCE
   - LLM used strategically (only ~1% of requests)
   - Not "call LLM for everything"
   - Keeps costs low while maintaining reliability

"How do you handle UI changes?"

Step 1: Check DOM fingerprint (fast)
Step 2: Try rule-based extraction
Step 3: If confidence < 70%, try heuristic
Step 4: If confidence < 50%, use LLM
Step 5: Always save with confidence score

"Can this scale?"

Yes. Designed for production:
- Handles 100-1000 cards/day (parallelizable)
- PostgreSQL backend for versioning
- Works with Airflow, Celery, cron
- Redis caching for fingerprints
- Health metrics & anomaly alerts
